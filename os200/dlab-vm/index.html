OS200 Lab On VirutalBox

This extra-credit lab teaches you how to recreate the OS200 lab environment on VirtualBox - so it's permanently available to you for experimentation. The instructions are for VirtualBox, but the golden image is VMDK format, so any hypervisor that supports VMDK can be used - such as VMware.

Step 1: Download and install Oracle VirtualBox

  1. Refer to these instructions to download your platform specific installer from Oracle and install and test VirtualBox.
  2. Create (or make sure you have) three host-only networks available

Step 2: Download the golden-image

  1. Download the golden dlab image: https://s3-us-west-1.amazonaws.com/dlab-togo/dlab-liberty-gold.vmdk
  2. Create a Linux/Ubuntu(64-bit) VM in virtualbox with: 2 vcpu 6G ram. Use the downloaded image as the VM's boot disk.
  3. Modify the networks of the new VM to have 3 Adapter/NIC - in hostonly interface mode.
    Note: You need 3 distinct hostonly interfaces. By default VirtualBox only has 1 hostonly interface defined. To create additional hostonly interfaces, using the VirtualBox Management Console, you can navigate to File/Preferences, select Network, select the "Host-only Networks" tab, and click the + button to create more networks.
  4. Power on and start with the OS200 Lab Instructions. Login credentials are the same as OS200 lab: stack/b00tcamp
  5. Since we do a lot of "sudo" and "sudo -i" optionally enable passwordless sudo for user stack, to save on typing:
    • Make sure you have two ssh windows into your machines - logged in as user "root" in one - because if you mess this up there is no easy way to recover root privilege - except through your root ssh window
    • On the first ssh window - become root by executing "sudo -i" and entering b00tcamp as the password
    • On the second ssh window execute "sudo vi /etc/sudoers" and modify /etc/sudoers as follows:
      Search for "/^sudo" and add NOPASSWD: before the last ALL - the line should look like this:
      %sudo   ALL=(ALL:ALL) NOPASSWD:ALL
    • Save the file and quit the editor. In the same window try "sudo -i". If it works you are done and can close the second window.
    • If sudo fails then use the second window to revert your changes and try again.
